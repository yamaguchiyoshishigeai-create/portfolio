Work Detail / Design Tool

APIM for Spring

Spring Frameworkシステム API・MCP設計支援ツール

自由文またはチェック式入力から、Spring向けREST API設計、MCP設計、外部AI投入用プロンプト、AI実装指示書のたたき台を整理する設計支援Webツールです。

Live Demo GitHub Worksへ戻る

Purpose

要件から設計への初動を、確認しやすい雛型へ変換する。

制作背景

業務要件からAPI設計へ落とし込む初動では、確認項目の抜けや設計粒度のばらつきが起こりやすくなります。APIMは、要件確認からREST API設計、MCP設計、外部AIへ手動投入するための安全なプロンプト、AI実装指示へ進む流れを一つのWebツールで整理するために制作しました。

解決したい課題

設計のたたき台がない状態では、関係者間の確認が抽象的になり、実装前の認識合わせに時間がかかります。自由文入力とチェック式入力から成果物を生成することで、議論の開始点を具体化し、レビューしやすい設計材料を用意します。

設計判断

自由文入力とチェック式入力を併用し、外部AI連携は、ユーザーが内容を確認したうえで外部AIへ手動投入できるプロンプト生成導線として扱います。人間がレビューできる設計成果物と、AI実装相談へ渡す入力を分けて整理することを重視しています。

Business Value

採用担当者・面談担当者向けの評価ポイント

業務要件を実装へ進める前段階で、API設計、MCP設計、対応表、Controller雛形、外部AI投入用プロンプト、AI実装指示書を一連の成果物として整理する制作物です。

要件から設計への初動を具体化

抽象的な要件確認で止まりやすい初期段階を、自由文入力またはチェック式入力によりAPI設計のたたき台へ変換します。

人間レビューを前提にした生成

自由文入力とチェック式入力を併用しながら、レビュー担当者が不足点や過剰な前提を確認しやすい成果物形式を重視しています。

AI実装支援への接続

API/MCP対応表、外部AI投入用プロンプト、AI実装指示書を出力対象に含め、設計判断を人間が確認したうえでAI実装相談へ渡せる流れを作っています。

Features

主な機能と技術構成

API設計書生成

自由文入力またはチェック式入力をもとに、Spring向けREST API設計のたたき台を生成し、エンドポイントや責務を確認しやすくします。

MCP設計書生成

API利用やAI連携を見据えたMCP設計の初期案を出力し、人間が確認する設計情報とAI利用前提の情報を整理します。

API/MCP対応表とAI実装支援

API設計、MCP設計、Spring Controller雛形、外部AI投入用プロンプト、AI実装指示書をつなげ、設計から実装相談へ進みやすい形式にします。

技術スタック

Java 17 / Spring Boot / Thymeleaf / Maven Wrapper / Markdown / MCP Design / Render を中心に構成しています。

Roadmap

今後の拡張方針

入力例と外部AIブリッジ説明の拡充

自由文入力とチェック式入力の利用例を増やし、ユーザーが内容を確認して外部AIへ手動投入できるプロンプト生成導線であることを分かりやすく説明します。

設計成果物の構造改善

生成されるAPI設計、MCP設計、対応表の粒度と構造を見直し、レビュー担当者が不足点、過剰な前提、実装前の確認事項を見つけやすい形に改善します。

生成プロンプトとAI実装指示書の改善

人間が確認した設計結果を、外部AIへ手動投入するプロンプトやAI実装指示書としてレビューしやすく整理し、設計判断と実装作業の接続を強化します。

# 運用改善ログ

このディレクトリは、portfolioリポジトリにおける軽量エラー履歴の受け口です。

## 目的

- 公開ポートフォリオ作業中の小規模な運用ミス、再発防止メモ、検証時の注意点を軽量に記録する。
- 通常TSKとして個票化する前に、低コストで事跡を残す。
- 公開安全性方針に反しない範囲で、次回作業者が参照できる運用観測を残す。

## 正本参照

軽量エラー履歴の詳細ルール、ライフサイクル、TSK化判断、共通課題と個別課題の分離方針は、`yamaguchiyoshishigeai-create/chatgpt-ops-rules` を正本とする。

本リポジトリ側には詳細規程本文を重複保持しない。作業開始時は `chatgpt-ops-rules` の現行規程を確認する。

## 公開安全性

このリポジトリは公開ポートフォリオ用であるため、軽量ログにも以下を記録しない。

- 住所、電話番号、生年月日、年齢
- 履歴書原本、スキルシート原本
- 実企業名、顧客名、内部システム名
- 守秘義務に触れる詳細仕様
- APIキー、access token、Cookie、`.env`、秘密鍵、フル環境変数
- 長大ログ全文、不要な絶対パス

## 配置

- `作業エラー履歴.md`: portfolio作業で利用する軽量エラー履歴の初期ファイル

## テンプレート初期状態

この初期状態には、APIM、Q-Scout、その他リポジトリ固有の実ログを含めない。portfolio作業で必要になった時点から記録を開始する。

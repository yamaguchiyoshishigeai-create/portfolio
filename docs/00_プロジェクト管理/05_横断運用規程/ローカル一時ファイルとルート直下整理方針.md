# ローカル一時ファイルとルート直下整理方針

## 1. 目的

本書は、portfolioリポジトリでChatExec方式、ChatExec2方式、Codex、手動確認を行う際に発生するローカル一時ファイルと、リポジトリルート直下に置くファイルの扱いを定義する。

portfolioは公開ポートフォリオ用リポジトリであるため、作業効率よりも公開安全性、秘密情報の混入防止、閲覧者に不要な内部作業痕跡を残さないことを優先する。

---

## 2. 最優先原則

1. APIキー、アクセストークン、秘密情報、環境変数ファイル、秘密鍵、個人情報をコミットしない。
2. 住所、電話番号、生年月日、年齢、履歴書原本、スキルシート原本を配置しない。
3. 実企業名、顧客名、内部システム名、守秘義務に触れる詳細仕様を配置しない。
4. ChatExec2やCodexの実行結果ファイルを公開成果物として扱わない。
5. 一時ファイル整理のために、正規文書や公開ページの内容を弱めたり削除したりしない。

---

## 3. ルート直下に置いてよいもの

リポジトリルート直下に置いてよいものは、公開リポジトリとして必要な最小限の正規ファイルに限定する。

例:

- `README.md`
- `PROJECT_START_PROMPT_TEMPLATE.md`
- `.gitignore`
- `package.json` など実装・ビルドに必要な正規ファイル
- GitHub Actions、設定ファイル、ライセンスなどの公開管理ファイル

---

## 4. ルート直下に残さないもの

以下は原則としてルート直下に残さない。

- `ChatExec.result`
- `ChatExec2_*.result`
- `ChatExec2_*.summary.md`
- `ChatExec2_*.log`
- `ChatExec2_*.out`
- `ChatExec2_*.err`
- `CodexExec_*.result`
- ローカル確認用ログ
- 一時メモ
- ダウンロードした外部資料
- 秘密情報、個人情報、非公開情報を含む可能性のあるファイル

これらは必要に応じて `.gitignore` 対象とし、コミット対象から除外する。

---

## 5. CodexExec指示書の扱い

`CodexExec_*.md` は、単なる一時指示書として使う場合と、後続の作業証跡として保存する場合がある。

そのため、`CodexExec_*.md` を一律にignoreしない。保存が必要な場合は、公開安全性を確認したうえで、適切なdocs配下へ配置する。

公開安全性を確認できない場合はコミットしない。

---

## 6. ChatExec2スクリプトの扱い

ChatExec2スクリプトを作成する場合は、以下を守る。

1. JOB_ID付きの一意なファイル名を使う。
2. 再配布する場合は `R01`、`R02` などの版識別子を付ける。
3. 実行結果には `[OK]` または `[FAIL]` を明示する。
4. Windows向け `.bat` / `.cmd` では、成功・失敗にかかわらず結果末尾を表示し、ウィンドウを保持する。
5. 結果ファイルには秘密情報や不要な個人情報を含めない。
6. GitHub Issue / PRコメントへ投稿する場合は、サニタイズ済み要約に限定する。

---

## 7. 完了条件

一時ファイル整理作業は、以下を満たすまで完了扱いしない。

- 正規ファイルと一時ファイルの区分が明確であること。
- コミット対象に一時ログ、秘密情報、個人情報、非公開情報が含まれていないこと。
- `git status` 上で意図しない未追跡ファイルが残っていないこと。
- 公開ページ、README、docsの必要内容が弱められていないこと。

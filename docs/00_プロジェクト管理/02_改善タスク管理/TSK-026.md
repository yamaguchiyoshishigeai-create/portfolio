# TSK-026 レスポンシブ時の日本語見出し・本文折り返しを設計する

状態: 未解決
優先度: Medium
担当主体: ChatGPT(リポジトリ編集)
提案名: PORTFOLIO-RESPONSIVE-LINEBREAKS-TSK026-01
登録日: 2026-06-05

## 背景

トップページのValue Proposition見出し「業務改善・設計支援・品質レビューをつなぐエンジニア」は、PC最大幅では1行表示が望ましい一方、スマホ幅など幅が狭い環境では意味のまとまりに沿って適切に折り返す必要がある。

この問題は当該見出しだけでなく、portfolio全体の日本語見出し・リード文・カード文言にも共通する可能性がある。

## 課題

現状は、箇所ごとに `white-space: nowrap`、`span` 分割、メディアクエリ、自然折り返しが混在している。

そのため、画面幅によっては以下の問題が起こり得る。

- PC幅では余白があるのに不自然に折り返す。
- 狭い画面では意味のまとまりと異なる箇所で折り返す。
- 長い日本語見出しで行長・視線移動・カード配置のバランスが崩れる。
- 個別対応が増え、今後の文言修正時に同種の表示崩れが再発する。

## 対応方針

即時の個別CSS修正ではなく、portfolio全体の表示文言を対象に、レスポンシブ時の折り返し設計を整理する。

対象候補:

1. トップページHero見出し・リード文
2. Value Proposition見出し・説明文
3. Whyセクション見出し
4. Worksセクション説明文
5. Works詳細ページのBusiness Value見出し
6. Careerページの見出し・主要職務経歴カード
7. カード見出し・ボタン周辺の短文

## 実装検討観点

- PC幅では不要な折り返しを避ける。
- タブレット幅・スマホ幅では意味単位で自然に折り返す。
- `white-space: nowrap` の使用範囲を限定し、横スクロールを発生させない。
- 必要な箇所は文言を意味単位の `span` へ分割する。
- 共通ユーティリティクラスとして、見出し用・リード文用の折り返し制御を整理する。
- CSSの `clamp()`、`max-width`、メディアクエリの基準をページ横断で再確認する。
- 既存の `.balanced-copy`、`.top-hero-lead`、`.why-heading-copy`、`PORTFOLIO-VALUE-HEADING-NOWRAP` 管理ブロックとの重複・競合を整理する。

## 非対象

- 今回のTSK登録時点では、HTML/CSSの実装修正は行わない。
- スクリーンショットの差し替えは、本課題の実装修正後に必要性を判断する。
- 文章そのものの訴求軸変更は原則として行わない。

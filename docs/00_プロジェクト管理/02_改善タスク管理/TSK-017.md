# TSK-017 chatgpt-ops-rules 中枢参照導入

状態: 未解決
優先度: High
担当主体: ChatGPT(リポジトリ編集)
提案名: PORTFOLIO-CENTRAL-REF-TSK017-01

## 背景

`chatgpt-ops-rules` が共通運用ルールの正本として整備された。

`portfolio` は公開系・軽量運用の既存作業リポジトリであり、共通運用ルール本文を重複保持せず、`chatgpt-ops-rules` への中枢参照を導入する必要がある。

一方で、portfolio側には以下の個別リポジトリ固有情報を残す必要がある。

- 公開ポートフォリオとしての掲載方針。
- 個人情報・守秘情報・実企業名・顧客名を置かない公開安全性方針。
- GitHub Pages公開前提。
- HTML / CSS / JavaScriptによる静的サイト仕様。
- Works掲載方針。
- portfolio固有の改善タスク管理。

## 目的

`portfolio` に `chatgpt-ops-rules` への中枢参照を導入し、共通運用ルールとportfolio固有仕様の分離を明確化する。

## 想定作業

本個票は、個票先行main反映ゲートに従い、本体作業前に先行してmainへ反映する。

本体作業では、以下を検討・反映する。

1. `PROJECT_START_PROMPT.md` を追加または既存入口を標準名へ整理する。
2. `README.md` / `docs/README.md` に `chatgpt-ops-rules` 参照導線を追加する。
3. `docs/00_プロジェクト管理/05_横断運用規程/` に軽量な中枢参照方針を追加する。
4. portfolio側に共通運用ルール本文を重複コピーしない。
5. portfolio固有の公開安全性方針・静的サイト仕様・掲載方針はportfolio側に残す。
6. ChatExec2を使う場合は、Python単一ファイル方式を標準候補として扱う。
7. 反映後に発生・残存課題の未個票化がないか確認する。

## 対象外

- 公開サイト本体のデザイン変更。
- Worksカードの追加・削除。
- GitHub Pages設定変更。
- 履歴書・スキルシート原本の配置。
- 個人情報や実企業名の追加。
- `chatgpt-ops-rules` 側の追加改定。

## 背景

`chatgpt-ops-rules` が共通運用ルールの正本として整備された。

`portfolio` は公開系・軽量運用の既存作業リポジトリであり、共通運用ルール本文を重複保持せず、`chatgpt-ops-rules` への中枢参照を導入する必要があった。

一方で、portfolio側には以下の個別リポジトリ固有情報を残す必要があった。

- 公開ポートフォリオとしての掲載方針。
- 個人情報・守秘情報・実企業名・顧客名を置かない公開安全性方針。
- GitHub Pages公開前提。
- HTML / CSS / JavaScriptによる静的サイト仕様。
- Works掲載方針。
- portfolio固有の改善タスク管理。

## 実施内容

1. `PROJECT_START_PROMPT.md` を追加した。
2. `README.md` / `docs/README.md` に `chatgpt-ops-rules` 参照導線を追加した。
3. `docs/00_プロジェクト管理/05_横断運用規程/chatgpt-ops-rules中枢参照方針.md` を追加した。
4. portfolio側に共通運用ルール本文を重複コピーしない方針を明確化した。
5. portfolio固有の公開安全性方針・静的サイト仕様・掲載方針をportfolio側に残す方針を明確化した。
6. ChatExec2を使う場合は、Python単一ファイル方式を標準候補として扱う方針を明確化した。
7. 改善タスク課題一覧を解決済みへ同期した。

# docs ガイド

この `docs` 配下は、ポートフォリオ制作・改善・公開運用に関する正本管理領域です。

## 基本方針

- 上位フォルダごとに責務を分ける。
- 公開サイト本体と、制作判断・要件・設計・改善タスクを分離する。
- 履歴書、スキルシート、住所、電話番号、生年月日、実企業名、顧客名、内部システム名などの非公開情報は配置しない。
- 公開文言では、実績を業界名・技術領域・担当工程へ抽象化する。
- 改善タスクは `TSK-###` 形式で採番する。

## 大分類

- `00_プロジェクト管理`: 方針、改善タスク、運用ルール
- `10_企画`: ポートフォリオの目的、対象読者、訴求軸
- `20_要件定義`: 掲載要件、公開/非公開境界、必要コンテンツ
- `30_基本設計`: サイト構成、情報設計、ページ構成
- `40_詳細設計`: セクション文言、データ構造、画面部品
- `90_アーカイブ`: 履歴保管

## 初期対象

初期版は、HTML / CSS / JavaScript による1ページ完結型の静的サイトとし、GitHub Pagesでの公開を想定します。

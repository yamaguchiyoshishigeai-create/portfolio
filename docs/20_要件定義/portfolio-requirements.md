# ポートフォリオ要件定義

## 機能要件

- 公開トップページを1ページ構成で提供する。
- Profile、Strengths、Skills、Projects、Career、AI/Web Learning、Contactを掲載する。
- Projectsは代表案件を公開向けに抽象化して掲載する。
- Skillsは言語、DB、OS、工程、業務領域に分類する。
- Contactは公開可能な導線のみを扱う。

## 非機能要件

- GitHub Pagesで公開可能な静的ファイル構成とする。
- 外部ビルドを不要とする。
- スマートフォン、タブレット、PCで閲覧しやすいレスポンシブ構成とする。
- 日本語で読みやすい正式なビジネス文体を基本とする。

## 公開制約

掲載しない情報:

- 住所
- 電話番号
- 生年月日
- 年齢
- 履歴書原本
- スキルシート原本
- 実企業名
- 顧客名
- 内部システム名
- 守秘義務に触れる詳細仕様

## 初期掲載プロジェクト候補

1. 医薬品業界向けEDI・受発注データ連携システム
2. 製造業向け工程制御・検査・解析システム
3. PaaSデータストア基盤開発
4. 金融機関向け基幹システム改修・運用支援
5. 組込み機器向けファームウェア開発
6. AI/Web学習成果物
